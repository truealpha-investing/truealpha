Methodology

True Alpha tracks publicly stated creator predictions over time and compares outcomes to the S&P 500 (SPY). This page documents the rules used to log and evaluate predictions. Not financial advice.

1) What counts as a prediction

Predictions are extracted from YouTube video transcripts using automated parsing and language models. A transcript mention is logged as a prediction when:

  • A ticker/asset is mentioned (e.g., TSLA, NVDA, BTC)
  • A directional claim is present (up/down/outperform/underperform)
  • A timeframe is stated or can be mapped into a standard bucket

Transcript-based extraction may contain errors. We continuously improve filtering and parsing rules.

2) Entry price rule

We log the entry price as the close price on the video publish date.

This is a modeling assumption and may differ from a viewer’s real execution price.

3) Horizon buckets (90 / 180 / 365)

Each prediction is evaluated at the end of its stated timeframe. If the creator is vague (e.g., “over the next few months”), we map it into one of these standardized horizons:

Short
90 days
Medium
180 days
Long
365 days

If a creator states a specific date, we use that date. Otherwise we map to the nearest bucket.

4) Benchmarking (Alpha vs SPY)

We compare the asset’s return to SPY over the same horizon. Alpha is:

Alpha = (Asset Return %) - (SPY Return %)

5) Leaderboard metrics

  • Average Alpha: average alpha across scorable predictions.
  • Win Rate (directional): % of predictions where direction was correct.
  • Sample size (N): number of scorable predictions included.

For public views marked “Verified”, we require N ≥ 20.

6) Limitations

  • Transcript extraction and interpretation can be imperfect.
  • Creators may change views over time; results update as new videos are added.
  • This dataset does not represent any creator’s full portfolio or intent.
