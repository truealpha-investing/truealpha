Methodology

True Alpha tracks publicly stated creator predictions over time and compares outcomes to the S&P 500 (SPY). This page documents the rules used to log and evaluate predictions. Not financial advice.

1) What counts as a prediction

Predictions are extracted from YouTube video transcripts using automated parsing and language models. A transcript mention is only logged as a prediction when:

  • A ticker/asset is mentioned (e.g., TSLA, NVDA, BTC)
  • A directional claim is present (up/down/outperform/underperform)
  • A timeframe is stated or mapped into a horizon bucket

Casual mentions without an implied outcome are excluded where possible, but transcript-based extraction can still contain errors.

2) Entry price rule

To keep the rule simple and repeatable, we log the entry price as the close price on the video publish date.

This is a modeling assumption and may differ from a viewer’s real execution price.

3) Horizons

Each prediction is evaluated at the end of its stated time horizon. Internally, horizons are bucketed as:

  • Short: < 6 months
  • Long: ≥ 6 months

If a creator states a specific target date, we use that date. If the timeframe is vague, we map it to a bucket.

4) Benchmarking (Alpha vs SPY)

Raw returns are compared against the S&P 500 using SPY over the same horizon. Alpha is calculated as:

Alpha = (Asset Return %) - (SPY Return %)

If a creator’s pick is up 5% but SPY is up 10% over the same period, alpha is -5%.

5) Leaderboard metrics

  • Average Alpha: average alpha across scorable predictions in the window.
  • Win Rate (directional): % of predictions where the direction was correct.
  • Sample size (N): number of scorable predictions included.

For public views marked “Verified”, we require N ≥ 20.
