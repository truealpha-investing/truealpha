Methodology

True Alpha tracks publicly stated creator predictions over time and compares outcomes to the S&P 500 (SPY). This page documents the rules used to log and evaluate predictions. Not financial advice.

1) What counts as a prediction

Predictions are extracted from YouTube transcripts using automated parsing and language models. A transcript mention is logged as a prediction only when it contains:

  • A ticker/asset reference (e.g., TSLA, NVDA, BTC)
  • A directional claim (bullish/bearish, up/down, outperform/underperform)
  • A timeframe (explicit) or enough context to map into a standard horizon

Transcript extraction and interpretation can be imperfect. We prioritize consistent rules over perfect parsing.

2) Entry price rule

We log the entry price as the close price on the video publish date.

This is a modeling assumption for repeatability. A viewer’s real execution price may differ.

3) Time horizons (90 / 180 / 365)

Every prediction is evaluated at a fixed horizon so results are comparable across creators. If a creator gives an explicit timeline, we map it into the nearest bucket:

  • Short: 90 days
  • Medium: 180 days
  • Long: 365 days
Example mapping:
“Over the next few months” → 90d
“Over the next 6 months” → 180d
“Over the next year” → 365d

4) Benchmarking (Alpha vs SPY)

Raw returns are compared against SPY over the same horizon. Alpha is:

Alpha = (Asset Return %) - (SPY Return %)

If a creator’s pick is up 5% but SPY is up 10% over the same period, alpha is -5%.

5) Leaderboard metrics

  • Average Alpha: average alpha across scorable predictions in the rolling window.
  • Win Rate: % of predictions where the direction was correct.
  • Sample size (N): number of scorable predictions included.

For public “Verified” views we require N ≥ 20.

6) Limitations

  • Transcript extraction can miss nuance or sarcasm.
  • Creators may change views; the rolling window reflects updates over time.
  • This does not represent a creator’s full portfolio or risk management.
